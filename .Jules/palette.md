## 2024-05-24 - Accessibility Fallbacks for Compact Forms Without Explicit Labels
**Learning:** The application heavily relies on "compact" UI designs where form inputs (like search bars, quick-reply textareas, and simple price inputs) use only `placeholder` attributes instead of explicit, visible `<label>` elements. While aesthetically clean, `placeholder` is not reliably announced by all screen readers as an accessible name, leaving visually impaired users without context. This is a recurring pattern across the public tracking pages and admin dashboards.
**Action:** When working on this repository, strictly enforce the use of `aria-label` attributes on any `input`, `textarea`, or `select` element that lacks an associated `<label>` to ensure baseline accessibility without breaking the intended compact visual design.

## 2024-06-21 - Visual Feedback and Duplicate Submission Prevention for File Upload Forms
**Learning:** The primary form includes an optional image upload that can take time to process, especially over slower connections. Because the form submission triggered no visual feedback, users could become impatient and click the submit button multiple times, potentially creating duplicate orders and uploads.
**Action:** When adding submit handlers to forms that upload files or perform slow operations, include inline JavaScript within the `onsubmit` handler (e.g., `setTimeout`) to disable the submit button, add a loading spinner, and change the button text to prevent duplicate clicks while providing immediate visual feedback to the user. Always use `setTimeout` with a 0ms delay to allow standard HTML form submission to continue uninterrupted after the UI changes.
