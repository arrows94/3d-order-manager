## 2024-05-24 - Form Field Accessibility Pattern
**Learning:** Found a recurring pattern in the app's forms (e.g., `index.html`, `admin_login.html`) where `<label>` elements were missing the `for` attribute and input fields lacked corresponding `id` attributes. Additionally, visual indicators for required fields were missing, making it difficult for both sighted and screen reader users to identify required information.
**Action:** When adding or modifying forms, always ensure each `<input>`, `<textarea>`, or `<select>` has an `id` that matches the `for` attribute of its corresponding `<label>`. For required fields, add a visual indicator like `<span aria-hidden="true" class="text-fuchsia-500">*</span>` so it's clear to sighted users but doesn't clutter screen reader output.
